# Powering the Future of AI: European Energy System Expansion Model

This repository contains the optimisation model and supporting material used in the research:

**"Powering the Future of AI Data Centres: Navigating the Trade-offs for Europe’s Energy Transition and Net-Zero Goals"**

The model analyses how rapid growth in **AI-driven data centre (DC) demand** affects the long-term evolution of the **European electricity system**, including generation expansion, transmission investment, and decarbonisation pathways.


# Data and Scripts Description

This repository contains several datasets and scripts required to reproduce the results of the study.

## 1. Renewable Availability and Demand Dataset

The file **`NewDataEurope.csv`** contains **hourly datasets for electricity demand and renewable resource availability** for **33 European countries** for the entire year **2024**.

The dataset includes:

- Hourly electricity demand
- Solar PV availability
- Onshore wind availability
- Offshore wind availability

For some countries where demand data is originally reported at **15-minute or 30-minute intervals**, the values were **averaged to obtain hourly demand values**.

---

## 2. Representative Day Clustering

The script **`Clustering_EU.py`** is used to reduce the temporal dimension of the dataset.

By executing this script, the user can:

- Select any desired number of **representative days**
- Reduce the full **8760-hour dataset** to a smaller set of **representative days**

When the script is executed:

- The renewable availability profiles and electricity demand profiles are clustered.
- **12 representative days** are generated by default.
- Each representative day is associated with a **weight factor**, indicating how many real days it represents.

The processed dataset containing the **representative days and their corresponding weights** is automatically saved and later used as input for the optimisation model.

---

## 3. Model Input Dataset

The file **`Input_Final.xlsx`** contains the **complete dataset required to run the optimisation model**.

This file includes:

- Technology cost parameters
- Operational parameters
- Fuel prices
- Emission factors
- Renewable land availability
- Transmission parameters
- Demand parameters
- Data centre parameters

All parameters and data sources used in this file are **fully documented with references** within the dataset.

---

## 4. Baseline Energy System Model

The script **`EUPan BaseModel`** implements the **baseline European energy system optimisation model** **without data centre demand**.

This model:

- Optimises generation capacity expansion
- Determines optimal operation of generation and storage
- Models cross-border electricity transmission

After execution, the model saves the results in **output files containing detailed system results**, including:

- Capacity expansion results
- Electricity generation
- Transmission flows
- System costs
- Emissions

---

## 5. Energy System Model with Data Centre Demand

The script **`EUPan Model with DC`** extends the baseline model by incorporating **additional electricity demand from data centres**.

This model evaluates **21 different data centre growth scenarios**.

The input assumptions include:

- Current installed data centre capacity
- Scenario-specific future capacity targets
- Country-level allocation of data centre demand

The scenario-specific constraints are implemented directly in the model code.  
Users can activate the desired scenario by selecting the corresponding constraint in the script. Detailed explanations are provided within the code.
The model can be run using either of the two load profiles provided in the input data file. Depending on the profile selected, the user should reference the corresponding column, as specified in the ‘DC’ sheet of the input file.

Because the **planned transmission expansion in Europe may not be sufficient to accommodate high data centre demand growth**, the **upper bound for technology build-out** in high-growth scenarios is relaxed to **120% of the baseline TYNDP build-out targets**.

When the model is executed:

- Each scenario produces a separate set of output results.
- Results are automatically stored in output files.

The framework is formulated as a ** Mixed-Integer Linear Programming (MILP) model** that co-optimises:

- Electricity generation capacity expansion  
- Cross-border transmission investments  
- Energy storage deployment  
- Data centre electricity demand growth  
- System reliability and carbon constraints  

The model is implemented using **Python and Pyomo**.

# Research Objective

The rapid expansion of artificial intelligence infrastructure is expected to significantly increase electricity demand from hyperscale data centres. This project investigates:

- How **AI-driven data centre demand growth** affects European power systems
- The **optimal spatial allocation of data centre capacity**
- The **impact on renewable integration and grid expansion**
- The **trade-offs between energy system costs, reliability, and decarbonisation**

The model integrates **data centre growth scenarios with long-term energy system planning**.

---

# Model Overview

The optimisation framework represents a **pan-European electricity system** with the following features:

### Spatial Resolution
- 33 European countries
- Explicit cross-border transmission representation

### Temporal Resolution
- Planning horizon: **2025–2050**
- **6 investment periods** (5-year steps)
- **Hourly operational resolution**
- Reduced to **12 representative days per year** via clustering

### Technologies Included

Generation technologies:

- Solar
- Onshore wind
- Offshore wind
- Nuclear
- Coal
- Gas (CCGT)
- Gas with CCS
- Hydrogen CCGT
- Biomass
- Biomass with CCS
- Oil

Energy storage:

- Battery storage
- Pumped hydro

Infrastructure:

- Cross-border transmission expansion

---

# Data Centre Modelling

The model explicitly represents **AI-driven hyperscale data centre electricity demand**.

Key modelling features:

- Initial European DC capacity (2024 baseline)
- Country-level spatial distribution
- Growth scenarios derived from:

  - IEA
  - ICIS
  - McKinsey

- **21 scenario pathways** describing different AI adoption trajectories
- Hyperscale load profiles based on **real operational data**

Up to **2030**, capacity growth is scenario-driven.  
After **2030**, optimal allocation of DC capacity is determined **endogenously by the optimisation model**.

---

# Objective Function

The optimisation minimises **total system cost**, including:

- Capital investment in generation and storage
- Transmission expansion costs
- Fixed O&M costs
- Variable generation costs
- Fuel consumption costs
- Carbon pricing costs
- Curtailment penalties
- Value of lost load (VOLL)

---

# Key System Constraints

The model includes several operational and planning constraints:

### Power system operation
- Hourly electricity balance
- Renewable generation availability
- Thermal ramping limits
- Curtailment limits

### Capacity expansion
- Technology build-rate limits
- Land availability for renewables
- Technology lifetime and decommissioning

### Transmission network
- Cross-border capacity expansion
- Transmission losses
- Bidirectional power flow limits

### Reliability
- Loss of Load Expectation (LOLE) constraint
- Maximum **3 hours/year** expected load shedding
- System-wide reserve margin (8%)

### Climate policy
- Carbon emission accounting
- Net-zero electricity sector target by **2050**

# Research Context

Digital infrastructure is becoming a major electricity consumer.  
AI-driven computing workloads may significantly increase:

- Peak electricity demand
- Infrastructure investment requirements
- Grid congestion risks

This research integrates **data centre expansion into continental-scale energy system modelling**, enabling evaluation of:

- Strategic infrastructure planning
- Energy transition trade-offs
- System-wide decarbonisation strategies.
